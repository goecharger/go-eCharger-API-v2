## Deutsch

1. Fehlerhafte Anzeige des MQTT-Verbindungsstatus behoben
2. Unzuverlässige MQTT-Verbindung behoben
3. Problem behoben, bei dem bei knappem Speicherplatz keine Offline-Daten mehr aufgezeichnet wurden
4. Falsch skalierte Temperaturmessgröße in OCPP behoben
5. Das Abbrechen des OTA-Firmware-Updates bei sehr langsamen Netzwerkverbindungen wurde behoben
6. Unterstützung für die Zeitplanung mit mehr als 2 Zeitfenstern pro Tag hinzugefügt 
7. Unterstützung für signierte Messwerte über OCPP für Kalibrierungsvorschriften basierend auf dem OCA-Standard
8. OCPP-Erweiterung hinzugefügt, um mehrere API-Schlüssel über GetConfiguration/ChangeConfiguration festzulegen/zu ändern
9. Debug-/Sudo-API-Schlüssel zum Auslösen des digitalen Ausgangs hinzugefügt
10. API-Schlüssel zum Speichern von Hardware-Revisionsinformationen hinzugefügt
11. API-Schlüssel zum Melden der Firmware-Version des Mobilfunkmodems hinzugefügt


## Englisch

1. Fixed incorrect MQTT connection state shown
2. Fixed unreliable MQTT connection
3. Fixed no more offline data being recorded when running out of disk space
4. Fixed incorrectly scaled temperature measurement measurand in OCPP
5. Fixed OTA firmware update being aborted on very slow network connections
6. Added support for scheduling with more than 2 time windows per day
7. Added support for signed meter values via OCPP for calibration law based on OCA standard
8. Added OCPP extension to set/change several API keys via GetConfiguration/ChangeConfiguration
9. Added debug/sudo API key to trigger digital output
10. Added API key to store hardware revision information
11. Added API key to report cellular modem firmware version
